辛扭转映射的 KAM 迭代算法

在一个动力系统的相空间中,拟周期轨道一般是指这样的不变子集: 其在拓扑上与一个 nnn 维环面同胚, 在动力学上, 往往存在着该环面上的某个参数化, 使得系统的动力学限制在环面上表现为简单的平移(针对映射系统)或者是线性流(针对流系统). 一般而言, 环面可能在系统的扰动下得以保持, 但环面上的流在扰动下可能会展现出极其复杂的性态.

在上述意义下, 拟周期轨道的存在性目前似乎只有唯一的研究工具: KAM 理论. 拟周期轨道的存在性往往可以归结为某个算子方程的解, KAM 理论通过线性化这个算子, 再利用频率的代数性质以及函数空间的性质, 往往可以求得这个线性化的算子的唯一解. 而后再精细地控制该解的定义区域以及残差项的误差, 通过迭代的方式得到原来非线性算子方程的精确解. 这个步骤包含了算子的线性化过程, 且一次迭代的残差项也是以上一次残差项的二阶小量的形式, 因此上述方法也称作是改进的牛顿迭代法. 几乎所有的 KAM 定理均涉及到了上述过程.

值得注意的是, 上述改进的牛顿迭代法完全可以在数值上进行实现, 并且实现的算法具有非常好的速度以及准确度, 同时该算法也能初步判断某个频率下环面是否破坏: 因为若该频率的拟周期解不存在, 算法是发散的. 下面我们就来详细介绍这种算法, 并具体地讨论其如何在一般的通有计算机语言上实现.

直接法

首先, 我们先要取定初始的 uuu 才能进行后续的计算. 一般地, 直接取 uuu 为平移即可, 即 u(θ)=θ+ωu(\theta)=\theta +\omegau(θ)=θ+ω. 我们将所要求解的式子展开为:

在数值上, 表示一个函数一般用等分布地 NNN 个点的函数值去近似一个函数. 由于我们所要处理的函数都是定义在 [0,1][0,1][0,1] 上, 故我们不妨取这 NNN 个点为 {0,1N,2N,...,N−2N,N−1N}\{0,\frac{1}{N},\frac{2}{N},...,\frac{N-2}{N},\frac{N-1}{N}\}{0,N1​,N2​,...,NN−2​,NN−1​}.

在一般的某一次迭代中, 我们仅仅知道函数 uuu 在这 NNN 个点上的分布. 要进行下一次迭代, 我们需要知道 vvv 在这 NNN 个点上的分布才行. 而上述方程的未知量似乎太多了: u−,u+,v−,v,v+u^-,u^+,v^-,v,v^+u−,u+,v−,v,v+. 实际上, 利用离散 Fourier 变换, 假设未知量是函数 vvv 在 NNN 个点的函数值, 那么 v−v^-v− 以及 v+v^+v+ 在这些点的函数值可以由 vvv 的函数值线性表出. 同样地, 可以利用离散 Fourier 变换求得 u−,u+u^{-},u^+u−,u+ 的分布, 那么上述方程中未知的仅有 vvv 了. 由于函数值有 NNN 个, 可以建立 NNN 个有 NNN 个未知数的方程. 注意到, 由于 v−v^-v− 以及 v+v^+v+ 的函数值可以由 vvv 的函数值线性表出, 这些方程是线性方程! 所以最后求 vvv 的函数值的问题变成求矩阵逆的问题.

这里我们总结一下离散 Fourier 变换及其逆变换的作用. 知道一个函数在区间上的等分布值, 我们可以利用离散 Fourier 变换求得其 Fourier 系数. 同样, 知道一个函数的 Fourier 系数, 并将其排列成上述特定的形式, 利用离散 Fourier 逆变换可以反求等分布的函数值.

具体算法的实现

直接法

我们再来回顾一下直接法需要做的事: 假设已知 uuu 在区间上的均匀分布, 我们需要求 vvv 在区间上的均匀分布, 而这两者由如下方程联系起来:

  • 假设函数 vvv 的等分布由 NNN 个未知量表出: {v0,...,vN−1}\{v_0,...,v_{N-1}\}{v0​,...,vN−1​}.

  • 计算函数 v−v^-v− 的分布, 通过以下方法.

    • 首先利用离散 Fourier 变换求得 vvv 的 Fourier 系数: vf={v0f,...,v−1f}v_f=\{v_{0f},...,v_{-1f}\}vf​={v0f​,...,v−1f​}.

    • 再将 vfv_fvf​ 的每个元素 vjfv_{jf}vjf​ 乘以 e−2πijωe^{-2\pi ij \omega }e−2πijω, 得到 vpfv_{pf}vpf​.

    • 再对 vpfv_{pf}vpf​ 做离散 Fourier 逆变换, 便得到了 v−v^-v− 的分布.

  • 同样方法计算 v+v^{+}v+ 的分布, 不过进行 Fourier 变换后要乘以系数 e2πijωe^{2\pi ij \omega }e2πijω.

  • 同样方法计算 u−,u+u^{-}, u^{+}u−,u+ 的分布.

  • 在 [0,1][0,1][0,1] 上的每个分布点线性化方程都成立. 对上述方程的每个分布点建立方程, 得到 NNN 个方程.

  • 由于只有 {v0,...,vN−1}\{v_0,...,v_{N-1}\}{v0​,...,vN−1​} 个未知量. 求解上述线性方程组得到 vvv.

  • 令新的 uuu 为 u+vu+vu+v. 循环实现一次.

  • 重复上述循环.

程序实现

同调方程 Julia 代码:

using FFTW
const n = 2^12
const ω = 2 \ (sqrt(5) - 1)
const k = 0.6

using Symbolics

@variables k s0 s1

hh = 0.5 * (s1 - s0)^2 - ((2 * π)^2 \ k) * cos(2 * π * s0)

hh1 = Symbolics.derivative(hh, s0)
hh2 = Symbolics.derivative(hh, s1)
hh12 = Symbolics.derivative(hh1, s1)

function mybuild(x)
    aa = build_function(x, k, [s0, s1])
    eval(aa)
end

const h, h1, h2, h12 = mybuild.([hh, hh1, hh2, hh12])

function e(x0, x1, x2)
    h2(k, [x0, x1]) + h1(k, [x1, x2])
end

function getp(x...)
    u, up, du, dup = x
    1 / (h12(k, [u, up]) * du * dup)
end


function gen_solvem()
    a = [1 / (1 - exp(-2 * pi * ω * im * i)) for i in -n/2:-1]
    b = [1 / (1 - exp(-2 * pi * ω * im * i)) for i in 1:n/2-1]
    fftshift([a; [0]; b])
end


function gen_solvep()
    a = [1 / (exp(2 * pi * ω * im * i) - 1) for i in -n/2:-1]
    b = [1 / (exp(2 * pi * ω * im * i) - 1) for i in 1:n/2-1]
    fftshift([a; [0]; b])
end

gen_solvep()

##

const transcoeff1 = fftshift([exp(2 * pi * ω * im * i) for i in -n/2:(n/2)-1])
const transcoeff2 = fftshift([exp(-2 * pi * ω * im * i) for i in -n/2:(n/2)-1])
const dcoeff = fftshift([2 * pi * im * i for i in -n/2:(n/2)-1])
const dcoeffp = fftshift([2 * pi * im * i * exp(2 * pi * ω * im * i) for i in -n/2:(n/2)-1])
const solvem = gen_solvem()
const solvep = gen_solvep()
const id = [i / n + 0 * im for i in 0:n-1]
const P = plan_fft(id)
const iP = plan_ifft(id)


function pm(x, y, μ)
    c = similar(x)
    for i in eachindex(c)
        c[i] = x[i] * (y[i] + μ)
    end
    c
end

function iter!(u0v, u0f)
    u0pwf = transcoeff1 .* u0f
    u0pwf[1] = u0pwf[1] + n * ω
    u0pwv = iP * u0pwf
    u0mwf = transcoeff2 .* u0f
    u0mwf[1] = u0mwf[1] - n * ω
    u0mwv = iP * u0mwf
    du0f = dcoeff .* u0f
    du0v = iP * du0f
    du0pwf = dcoeffp .* u0f
    du0pwv = iP * du0pwf
    gv = -(1 .+ du0v) .* map(e, u0mwv + id, u0v + id, u0pwv + id)
    gf = P * gv
    ψv = iP * (gf .* solvem)
    pv = map(getp, id + u0v, id + u0pwv, 1 .+ du0v, 1 .+ du0pwv)
    μ = -sum(pv .* ψv) / sum(pv)
    wf = solvep .* (P * (pm(pv, ψv, μ)))
    wv = iP * (wf)
    preu0v = u0v + ((1 .+ du0v) .* wv)
    preu0f = P * (preu0v)
    for i in eachindex(preu0v)
        u0v[i] = preu0v[i]
        u0f[i] = preu0f[i]
    end
end

function runcode()
    u0v = [ω + 0im for i in 1:n]
    u0f = P * u0v
    for i in 1:100
        iter!(u0v, u0f)
    end
    u0v, u0f
end

@time aa = runcode();
